# 美股财报查看网站需求文档

## 1. 背景与目标

用户会长期关注并购买美股，希望有一个轻量网站，用来选择公司并查看指定年度或季度的财报。网站第一版聚焦美国上市公司 10-K/10-Q 财务数据，优先服务个人投资者的快速阅读、对比和理解。

第一版目标：

- 支持常见美股公司选择，例如英伟达、特斯拉、苹果、微软、亚马逊、Alphabet、Meta。
- 支持选择财年和季度，查看该期核心财报数据。
- 支持中英对照展示，让用户能同时看到英文原始会计科目和中文释义。
- 提供自动化财报解读，总结收入、利润、现金流、资产负债表和风险信号。
- 数据来源优先使用 SEC EDGAR/XBRL 公开数据，保证可追溯和可扩展。

## 2. 用户与使用场景

主要用户是个人美股投资者。典型场景：

- 买入前快速查看一家公司的最近季度收入、利润、现金流和负债情况。
- 对某一年 10-K 或某个季度 10-Q 做粗读，先看机器整理后的重点，再决定是否阅读原始公告。
- 在公司之间切换，形成对不同股票财务质量的初步判断。

## 3. 功能范围

### 3.1 公司选择

- 内置公司清单：NVDA、TSLA、AAPL、MSFT、AMZN、GOOGL、META。
- 每家公司显示中文名、英文名、股票代码。
- 支持输入美股 ticker，从 SEC ticker-CIK 映射中解析公司并加载财报。
- 后续可扩展为收藏列表和自定义公司分组。

### 3.2 年度与季度选择

- 支持选择财年。
- 支持期间选择：年度 FY、季度 Q1/Q2/Q3。
- 第一版直接展示 SEC 已披露的 10-K/10-Q 数据。Q4 在美股披露中通常包含在年度 10-K 里，不单独作为 10-Q 展示。
- 显示每个公司对应 FY/Q1/Q2/Q3 的实际覆盖日期。由于美股公司财年不一定等于自然年，季度按钮和报表标题下方都应标注起止日期。
- Q4 通常不单独披露 10-Q，而是包含在 10-K 年报中；当 FY 和 Q3 日期可推导时，FY 旁应标注隐含 Q4 的覆盖日期。

### 3.3 财报展示

- 展示三类核心报表：
  - 利润表：收入、毛利、营业利润、净利润、研发费用、销售管理费用、稀释 EPS 等。
  - 资产负债表：现金、应收账款、存货、总资产、总负债、股东权益、长期债务等。
  - 现金流量表：经营现金流、资本开支、自由现金流、融资现金流等。
- 每一项展示：
  - 中文名称
  - 英文 XBRL/财务科目名称
  - 数值
  - 单位
  - 截止日期
  - 申报表类型
- 支持缺失项提示。不同公司披露口径会有差异，缺失不等于公司没有该项目。
- 支持从工作台直接跳转到 SEC 对应公司的对应年度/季度原始 10-K/10-Q 财报。

### 3.4 中英对照

- UI 默认显示中文名称和英文名称。
- 英文名称采用 SEC XBRL tag 或标准财务科目名，便于用户与原始 filings 对照。
- 后续可扩展为语言开关、术语词典和原文摘录。

### 3.5 智能解读

第一版使用规则引擎解读，避免引入账号、密钥和大模型成本。解读内容包括：

- 收入、利润、毛利率、经营利润率、净利率。
- 经营现金流与自由现金流质量。
- 资产负债结构，例如资产、负债、权益和负债率。
- 重点提醒：亏损、自由现金流为负、负债率偏高、关键项目缺失。

后续版本可以接入 LLM，把结构化数据、同比/环比变化和原始公告片段送入模型，生成更接近研究员风格的长文解读。

### 3.6 数据对比

- 支持在同一家公司内对比不同年度或季度。
- 支持对比指标：营业收入、净利润、自由现金流、毛利率、负债率、总资产、总负债。
- 支持选择年度 FY 序列或季度序列，展示最近 4、8、12 期。
- 使用柱状图和表格同时展示对比结果，方便快速观察趋势和异常值。

## 4. 数据来源与技术约束

数据源为 SEC EDGAR Data APIs：

- `https://data.sec.gov/api/xbrl/companyfacts/CIK##########.json`
- 官方文档：`https://www.sec.gov/edgar/sec-api-documentation`
- 数据格式为 XBRL facts JSON。
- 该接口不支持浏览器 CORS，因此网站需要本地后端代理请求 SEC。
- 后端请求 SEC 时应带 `User-Agent`，可通过 `SEC_USER_AGENT` 环境变量配置。

第一版不是审计工具，不保证覆盖所有原始财报明细。它把 SEC XBRL 中常用、可比性较强的 US-GAAP concepts 抽取成投资者阅读视图。

## 5. 非功能需求

- 响应速度：同一家公司数据在服务端内存缓存，减少重复访问 SEC。
- 可维护性：公司清单、财报项目映射、解读规则分离。
- 可用性：网络失败、SEC 无数据、某期缺失项目时给出明确提示。
- 视觉风格：偏投资工作台，信息密度适中，避免营销式落地页。

## 6. MVP 验收标准

- 启动本地服务后，用户能打开网站并选择公司。
- 用户能输入 SEC 覆盖的美股 ticker 并切换到该公司。
- 用户能选择年度和 FY/Q1/Q2/Q3。
- 用户能看到所选公司每个季度具体覆盖的起止日期。
- 点击加载后，页面展示三张核心报表的中英对照数据。
- 用户能从当前报表跳转到 SEC 原始 filing 页面。
- 用户能选择指标和范围，对比不同年度或季度数据，并看到图形化展示。
- 页面展示一段结构化智能解读。
- 网络或数据缺失时，页面不会崩溃，并能提示原因。

## 7. 后续迭代

- 展示同比、环比、趋势图。
- 链接到 SEC 原始 filing 页面。
- 接入 OpenAI 等模型做深度财报问答。
- 增加估值指标、市值、股价和分析师预期。
- 增加关注列表、财报日历和提醒。
